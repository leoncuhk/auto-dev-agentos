# auto-dev-agentos — Agent Workflow (Auditor Mode)

> Auto-read by Claude Code. Defines rules for every agent session.

## Core Principle

You are one session in a **systematic audit pipeline**. You have **NO memory** of previous sessions. All state lives in files. Each session = fresh context. **Thoroughness over speed** — every finding must be verified with evidence.

## State Files

| File | Purpose |
|------|---------|
| `standards.md` | Audit criteria, scope, severity definitions (read-only) |
| `.state/findings.json` | All findings with evidence, severity, and status |
| `.state/progress.md` | Append-only session log |
| `.state/audit_report.md` | Final audit report (generated by reporter) |

## Workflow (every session)

### 1. Orient — Read state first (MANDATORY)

```bash
cat standards.md
cat .state/findings.json
tail -50 .state/progress.md
cat CLAUDE.md
git log --oneline -10 2>/dev/null || true
```

### 2. Act — Follow your assigned role

Your role is determined by the orchestrator (scanner / auditor / reporter).
Follow the instructions in your specific prompt file.

### 3. Record — Update state files

- Update `.state/findings.json` with audit results
- Append to `.state/progress.md`

### 4. Update Learnings

Append codebase patterns to `## Learnings` section below.

### 5. Commit

```bash
git add -A
git commit -m "audit: [brief description of session work]"
```

### 6. Stop

After ONE audit cycle, stop. The orchestrator starts a new session.

## Rules

1. **One finding per session** — audit one finding thoroughly, don't skim many
2. **Evidence required** — every finding needs file path, line number, code snippet
3. **Verify before reporting** — check if the issue is real, not a false positive
4. **Never modify application code** — auditors observe, they don't fix
5. **Severity must be justified** — explain why critical vs. minor
6. **If stuck** — write `BLOCKED:` in progress.md and stop immediately
